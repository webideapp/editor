Streamlining Deployment with the New Terminal

The command line can be intimidating on mobile, but it's an essential tool for modern web development. With the latest update to Web IDE, we've reimagined the terminal experience to bridge the gap between complex Git operations and mobile-first usability.

The Intelligent Git Wizard

Gone are the days of manually typing long chains of Git commands on a virtual keyboard. We've introduced a smart Git Wizard that actively assists you when you're ready to deploy.

Simply type git push in the terminal, and the IDE takes over. If you haven't configured your remote repository yet, the wizard guides you through the entire lifecycle:

  • Initialization: Checks if a git repository exists, and creates one if needed.
  • Staging: Automatically adds your project files to the staging area.
  • Committing: Prompts for a commit message or uses a smart default.
  • Pushing: Uploads your code to the designated GitHub branch.

Security First: Access Tokens

Security is paramount in modern development. GitHub has deprecated password authentication in favor of more secure methods. Web IDE fully supports Personal Access Tokens (Classic).

Authentication Requirement

When prompted for a password in the terminal, you must use your GitHub Personal Access Token. Using your standard account password will result in an authentication failure.

You can generate a token in your GitHub Developer Settings. Ensure you check the repo scope to allow Web IDE to write to your repositories.

Deploying to GitHub Pages

The ultimate goal of many web projects is to be seen. The new terminal integration makes enabling GitHub Pages seamless.

As seen in the screenshot above, once your push is successful, the terminal provides direct feedback on your branch status (e.g., main -> main). From there, the process is simple:

  1. Go to your repository settings on GitHub.
  2. Navigate to the Pages section.
  3. Select your branch (usually main) as the source.
  4. Save, and your site is live on the web!

Why use the CLI on Mobile?

While GUI buttons are great, the terminal offers precision. You can see exactly what files are being staged and catch errors before they leave your device. It's professional-grade control, optimized for touch.
