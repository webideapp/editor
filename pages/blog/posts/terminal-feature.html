Streamlining Deployment with the New Terminal

The command line can be intimidating on mobile, but it's an essential tool for modern web development. With the latest update to Web IDE, we've reimagined the terminal experience to bridge the gap between complex Git operations and mobile-first usability.

The Intelligent Git Wizard

Gone are the days of manually typing long chains of Git commands on a virtual keyboard. We've introduced a smart Git Wizard that detects when you're trying to push code.

Simply type git push in the terminal, and the IDE takes over. If you haven't configured your remote repository yet, the wizard guides you through:

  1. Initializing the repository.
  2. Adding your files.
  3. Committing changes with a default or custom message.
  4. Pushing to your designated GitHub branch.

Security First: Access Tokens

Security is paramount. GitHub no longer accepts account passwords for command-line operations. Web IDE fully supports Personal Access Tokens (classic).

Note: When prompted for a password, you must use your GitHub Personal Access Token. Standard passwords will result in an authentication failure.

You can generate a token in your GitHub Developer Settings. Ensure you check the repo scope to allow Web IDE to write to your repositories.

Deploying to GitHub Pages

The ultimate goal of many web projects is to be seen. The new terminal integration makes enabling GitHub Pages seamless.

As seen in the screenshot above, once your push is successful, the terminal provides direct feedback on your branch status (e.g., main -> main). From there, you can hop over to your repository settings on GitHub and enable GitHub Pages to host your site live on the web in seconds.

Why use the CLI on Mobile?

While GUI buttons are great, the terminal offers precision. You can see exactly what files are being staged and catch errors before they leave your device. It's professional-grade control, optimized for touch.
